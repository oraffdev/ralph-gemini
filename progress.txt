## Codebase Patterns
- Pattern: Use 'files' property in tsconfig.json to specify entry points to avoid configuration conflicts.
- Pattern: FPS control in requestAnimationFrame is essential for a game like Snake.
- Pattern: Main game logic is in snake-game-js/main.ts, which is compiled to main.js.
- Pattern: Canvas size is 400x400 with a 20x20 GRID_SIZE.
- Pattern: Use keydown event listener to control the snake's direction.
- Pattern: Always check the current direction before updating to prevent 180-degree reversals.
- Pattern: Use `npx -y -p typescript tsc --project snake-game-js/tsconfig.json` for compilation if typescript is not locally installed.
- Pattern: Snake growth is achieved by omitting the 'pop' operation during a frame where food is eaten.
- Pattern: Use a `Point` type for coordinates to ensure type safety.
- Pattern: Always verify food placement against the snake's current positions to avoid unfair spawns.
- Pattern: Check for boundary and self-collision in the 'update' function before updating the snake's position.
- Pattern: Use a semi-transparent overlay in 'draw' to show a Game Over screen without clearing the final game state.
- Pattern: Restart the game using `location.reload()` when Space/Enter is pressed after Game Over.
